BACKEND (dms_backend):
  ├─ Function: calculateScheduledTime()
  │  ├─ Reads: settings.events[eventType]
  │  ├─ Logic:
  │  │  ├─ IF appointmentReminder:
  │  │  │  └─ scheduledAt = appointmentTime - hoursBeforeAppointment
  │  │  │
  │  │  └─ ELSE (all others):
  │  │     └─ scheduledAt = now + delayMinutes
  │  │
  │  └─ Returns: Date or null (immediate)
  │
  ├─ When event triggers:
  │  ├─ createAppointment()
  │  │  ├─ Queue: appointmentBooked (with calculateScheduledTime)
  │  │  └─ Queue: appointmentReminder (with calculateScheduledTime)
  │  │
  │  ├─ rescheduleAppointment()
  │  │  ├─ Queue: appointmentRescheduled
  │  │  └─ Queue: appointmentReminder (NEW time)
  │  │
  │  ├─ cancelAppointment()
  │  │  └─ Queue: appointmentCancelled
  │  │
  │  ├─ markTreatmentComplete()
  │  │  ├─ Queue: appointmentCompleted OR treatmentScheduled (MUTEX)
  │  │  └─ Trigger: postCare journey (each step with per-step delay)
  │  │
  │  ├─ createInvoice()
  │  │  └─ Queue: invoiceGenerated
  │  │
  │  └─ addPrescription()
  │     └─ Queue: prescriptionIssued
  │
  ├─ Function: triggerJourneyWithDynamicScheduling()
  │  ├─ For each step in journey:
  │  │  ├─ Calculate: scheduledAt = completedAt + step.delay.value
  │  │  └─ Queue: postCare message with that scheduledAt
  │  │
  │  └─ Example:
  │     ├─ Step 1: +1h delay → queue for completedAt+1h
  │     ├─ Step 2: +24h delay → queue for completedAt+24h
  │     └─ Step 3: +48h delay → queue for completedAt+48h
  │
  └─ Sends to WAAPI: { messageType, scheduledAt, idempotencyKey, ... }

    ⬇️

WAAPI:
  ├─ Receives: { scheduledAt: "2026-04-27T10:30:00Z", ... }
  ├─ Status: queued
  └─ At scheduled time: Sends to patient


💡 KEY INSIGHT: Two Types of Delays
═════════════════════════════════════════════════════════════════════════════

TYPE 1: BEFORE Event
  └─ appointmentReminder
     ├─ Config: hoursBeforeAppointment (default 24)
     ├─ Calculation: appointmentTime - (24 * 3600000 ms)
     ├─ Example: Appointment at 2PM → Send at 2PM-24h = prev day 2PM
     └─ ONLY uses hoursBeforeAppointment (not delayMinutes)

TYPE 2: AFTER Event
  ├─ All other message types:
  │  ├─ appointmentBooked
  │  ├─ appointmentRescheduled
  │  ├─ appointmentCancelled
  │  ├─ appointmentCompleted
  │  ├─ treatmentScheduled
  │  ├─ invoiceGenerated
  │  ├─ prescriptionIssued
  │  └─ postCare (each step)
  │
  ├─ Config: delayMinutes (default 0)
  ├─ Calculation: now + (delayMinutes * 60000 ms)
  ├─ Example: Invoice created at 10AM with 30min delay → Send at 10:30AM
  └─ For postCare: Uses step's own delay { value: X, unit: 'hours' }

🎓 KEY CONCEPTS TO UNDERSTAND
═════════════════════════════════════════════════════════════════════════════

1. Frontend Settings Structure:
   - Each event type has delay configuration
   - appointmentReminder is special: hours BEFORE (not delay after)
   - Everything else: minutes AFTER

2. Backend Calculation:
   - Fetch settings when event happens
   - Call calculateScheduledTime() to get scheduledAt
   - Pass scheduledAt to queueScheduledMessage()

3. WAAPI Integration:
   - scheduledAt is ISO8601 timestamp
   - null scheduledAt = send immediately
   - Future scheduledAt = queue and send later

4. MUTEX Group:
   - Lines 25 in WhatsAppPage.jsx
   - Only ONE can be active:
     [ appointmentCompleted, invoiceGenerated, invoiceAndPrescription,
       prescriptionIssued, treatmentScheduled ]
   - Check enabled status, only queue if enabled

5. PostCare Journey:
   - Multiple steps, each with own delay
   - Delays from TreatmentJourney document
   - triggerJourneyWithDynamicScheduling() loops through steps
   - Each step: scheduledAt = completedAt + step delay
