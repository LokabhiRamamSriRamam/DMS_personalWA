HOW IT WORKS
================================================================================

1. SETUP (One-time)
   └─ DMS: Clinic admin configures follow-up messages for each rating
   └─ DMS: Clinic admin configures poll question
   └─ WAAPI: Receives poll template (via DMS)

2. APPOINTMENT COMPLETION
   └─ User clicks "Conclude Appointment"
   └─ DMS sends poll to WAAPI
   └─ WAAPI delivers poll to patient via WhatsApp

3. PATIENT RESPONDS
   └─ Patient taps one of 5 rating options
   └─ WAAPI receives response via Baileys
   └─ WAAPI maps response index to rating (1-5)

4. AUTO-FOLLOW-UP
   └─ WAAPI calls DMS API to get follow-up message
   └─ DMS returns message for that rating
   └─ WAAPI queues and sends follow-up to patient

5. TRACKING
   └─ DMS can query poll responses from WAAPI
   └─ View aggregated feedback statistics
   └─ Filter by patient, rating, date range

DATA STRUCTURE
================================================================================

Follow-up Messages (stored in DMS)
──────────────────────────────────────────────────────────────────────────────
{
  tenantId: "clinic-001",
  feedbackType: "excellent",           // excellent, good, neutral, poor, very_poor
  rating: 5,                           // 1-5
  contentType: "text",                 // text, image, document, location
  content: { text: "Thank you..." },   // Flexible per contentType
  isEnabled: true,
  sendDelay: 0,                        // Milliseconds
  createdAt: Date,
  updatedAt: Date
}

Poll Responses (stored in WAAPI)
──────────────────────────────────────────────────────────────────────────────
{
  tenantId: "clinic-001",
  messageId: "msg-123",
  from: "919876543210",
  pollQuestion: "How satisfied are you?",
  selectedIndex: 3,                    // 0-4 for 5-option poll
  rating: 4,                           // Calculated: selectedIndex + 1
  feedbackType: "good",                // Mapped from rating
  createdAt: Date,
  updatedAt: Date
}
